用 GitHub Pages 发布演示（不用学 npm）

仓库已配置自动流水线：只要你把代码推到 GitHub，云端会自动执行构建并更新公开网页，链接可长期分享给同事，效果接近 Figma 的 Published / Share link。

你需要做的（只做一次）

  1. 在 GitHub 上新建一个空仓库，或把本仓库推上去。
  2. 打开该仓库网页 → Settings（设置）→ 左侧 Pages。
  3. 在 Build and deployment 里，Source 选 GitHub Actions（不要选 Deploy from a branch，除非你知道区别）。
  4. 回到仓库首页 → 顶部 Actions，确认名为 Deploy Feature Store (GitHub Pages) 的工作流在推送后出现；若失败，点进去看日志（一般是权限或分支名问题）。

演示网址长什么样

一般格式如下（把「用户名」「仓库名」换成你的）：

https://你的GitHub用户名.github.io/仓库名/

打开后就是最新的 React 交互原型。以后你改代码并 push 到默认分支（main 或 master），等几分钟流水线跑完，网页会自动更新。

什么时候会重新发布

当你推送的改动涉及 docs/references/feature-widetable-figma-export/ 里的文件时，会自动重新构建。也可在 Actions 里手动 Run workflow。
