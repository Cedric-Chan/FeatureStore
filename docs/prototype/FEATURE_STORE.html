Online Feature Store 交互演示

交互原型为 Vite + React（仓库内唯一真源）。

推荐：公开网页（GitHub Pages，类似 Figma Published，无需本地 npm）

把本仓库推到 GitHub 后，工作流会把 feature-widetable-figma-export 构建结果发布为固定公开网址，浏览器打开即可体验完整交互（WideTable、Feature Group、Feature Map 等）。

本仓库当前公开演示（可直接点开）：
https://cedric-chan.github.io/FeatureStore/

路由为前端 Hash（地址栏会带 #），例如 WideTable 列表 …/#/wt，Feature Group …/#/fg，Feature Map …/#/fm。

自己 fork/自建仓库时，如何找到对应的 Pages 地址：

  1. 打开 GitHub 上的本仓库 → 右侧或顶部 About 里若有 Website 链接，一般即 Pages 主页；或
  2. Settings → Pages，在部署成功后页面会显示站点 URL；或
  3. Actions → 工作流 Deploy Feature Store (GitHub Pages) → 最近一次成功运行 → 展开 deploy 步骤，环境信息里会带 page_url。

通用格式：https://<你的GitHub用户名>.github.io/<仓库名>/

首次启用 Pages、排查流水线：一步步说明

其他方式（可选）

  • 本地开发：进入 docs/references/feature-widetable-figma-export，执行 pnpm install 与 pnpm dev（需已 corepack enable）。
  • 离线 zip：同目录执行 pnpm run build，使用生成的 dist 打包分享。
  • 本文件夹预览：同目录执行 pnpm run sync-prototype 后，可打开 feature-store-demo/index.html（需本机先执行过同步）。
